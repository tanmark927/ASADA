What is this project?

ASADA is an Alexa Support Anti-Depression Application that aims to help alleviate people suffering from depression. The main purpose of this project is to help users cope with struggling with depression and lower their depression levels by talking to Alexa on a daily basis. It aims to give its user other things to do in order to take its depressed user to focus on their depression. These include many fun activities including a fortune-cookie event.

Features

Main Alexa Application:

  • Talk to Alexa about your day, and Alexa will respond based on responses
  • Take a Depression Index Quiz that gives a score of users' depression levels
  • Alexa will record responses to a database

Android Application (Currently in Development and not implemented):

  • View chat history logs
  • Refresh to up-to-date status of database

How to Install

Alexa Application:

  • Go to the menu and select Skills or go to the Alexa Skills store on the Amazon website
  • In the search bar, type in ASADA
  • Navigate down, and select the skill detail page.
  • Select the Enable Skill option and ask Alexa to open the skill
To manage your skills:
  • Go to the menu and select Skills
  • Select Your Skills at the top of the screen

Android Application:

  • Download the APK file from our github page onto your Android Device
  • ! Be sure you allow unknown apk files installation settings turned on !
  • Navigate to where you downloaded the file, and press on the file to begin installiation

Components and Dependencies

We used the Alexa Skill Kit to create the Alexa app. From there, we used the development GUI to create our intents and sample utterances. From there, we connected the Alexa app to an AWS Lambda function. In the Lambda function, we are programming with the Python language, along with PyMySQL to connect to our database. The database is built on Amazon RDS, since we wanted to stick with strictly Amazon Web Services. From our understanding, how the Alexa program works is that Alexa will listen to commands over voice, and figures out the utterance or intent on its own, and sends the intent command to Lambda. From Lambda, it will generate the response there and will access the database with our PyMySQL dependency. Then Lambda sends back the response of the utterance to Alexa, and Alexa will speak out the response. We also used Android Studio 3.1.1 to create our android app, which is based on Java and the Android development kit.

How to Contribute

You can contribute to this by going to our Github page.
We encourage that you create your own branch, and create any pull requests for us to review

The current issues can be viewed on our Issues page.


You first need an AWS developer account and an Amazon account. (they both have different set up processes)
You then need to set up instances of Lambda, and RDS. Set the constraints so that it is running on Linux and MYSQL instances for RDS.
You can copy over our SQL code from the github to build the RDS database. You can follow this tutorial to connect to your RDS database from mySQL workbench. Be sure to open port 3306 in your security groups.
Then, you need to create an Alexa Skill for the testing. Then you need to link both the Lambda function and The Skill. Amazon has a great tutorial for this. Tutorial
You can copy our JSON code to the Alexa Skill, and our Python code to the Lambda function.

If you do not want to follow the steps, we can provide AWS accounts on contact with us at asadadepression@gmail.com
But be warned, we do not give this out to anybody.

Frequently Asked Questions

How was the database connected to that application?

The lambda function connects to an ECS server in order to communicate with RDS.

I cannot connect to my RDS database. Help?!

You have to set the security group to accept inbounds and outbounds of port 3306. That way, MySQL will be able to connect

My Alexa function is failing to run

There is an error in the Lambda function. You need to check your code over, and ensure there is not any errors.
If you are confident there are no errors, check that your Lambda function is connected to your Alexa Skill
